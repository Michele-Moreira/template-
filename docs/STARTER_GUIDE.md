# Guia de Início

Use este template como base para criar projetos ágeis e consistentes.

## 1. Defina o propósito

- Hackathon: focar em MVP, protótipo funcional e apresentação.
- Estudo: priorizar aprendizado, clareza de código e documentação.
- Portfólio: mostrar valor, design e impacto com exemplos e capturas.

## 2. Mantenha a estrutura

- `src/` - código principal
- `tests/` - validações automáticas ou scripts de verificação
- `public/` - arquivos públicos, assets e demos
- `docs/` - notas, guias e decisões
- `assets/` - imagens, ícones e recursos visuais

## 3. Documente desde o início

- Atualize `README.md` com contexto, instruções e resultados esperados.
- Use `CONTRIBUTING.md` se outras pessoas participarem.
- Salve decisões importantes em `docs/PROJECT_STRUCTURE.md`.

## 4. Dados de referência

- Nome do projeto: escolha algo simples e memorável.
- Descrição curta: explique o que o projeto resolve.
- Tecnologias: liste as opções ou o stack escolhido.
- Status: `Rascunho`, `Em desenvolvimento`, `Concluído`.

## 5. Exemplo de checklist rápido

- [ ] Definir objetivo
- [ ] Criar estrutura de pastas
- [ ] Registrar dependências e comandos
- [ ] Criar README com instruções
- [ ] Adicionar testes básicos
- [ ] Registrar funcionalidades no `CHANGELOG.md`
