# Regras do repositório

- Preserve o caráter agnóstico do template; não assuma stack específica sem necessidade.
- Mantenha a documentação genérica e independente de qualquer IA, editor ou fornecedor específico.
- Faça mudanças pequenas e focadas no requisito atual.
- Não remova estrutura útil de documentação, histórico ou planejamento.
- Mantenha `README.md`, `CONTRIBUTING.md`, `CHANGELOG.md` e `ROADMAP.md` coerentes com as alterações.
- Prefira instruções curtas, claras e reutilizáveis.
- Se uma mudança afetar organização ou fluxo, reflita isso também em `docs/` quando fizer sentido.
- Evite duplicar regras entre arquivos; centralize aqui o que for operacional.
- Sempre use commits seguindo o padrão Conventional Commits
