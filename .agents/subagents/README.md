# Subagents

Use subagentes para acelerar tarefas que podem ser isoladas sem risco de conflito com o trabalho principal.

## Quando acionar

- Localizar arquivos ou padrões em uma base maior.
- Ler documentação ou código adjacente para montar contexto rápido.
- Investigar alternativas antes de editar.
- Checar áreas independentes em paralelo.

## Quando evitar

- Alterações pequenas e óbvias em um único arquivo.
- Fluxos que dependem de uma decisão sequencial imediata.
- Tarefas em que o custo de sincronização é maior que o ganho.

## Regra prática

- Use subagentes para descobrir e confirmar.
- Faça a edição principal no agente atual, com escopo mínimo.
