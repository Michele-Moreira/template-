# Contributing

Obrigado por usar este template! Aqui estão boas práticas para manter projetos consistentes, rápidos e fáceis de adaptar.

## Como usar

1. Abra uma nova branch com nome claro: `feature/<nome>`, `fix/<nome>` ou `hackathon/<nome>`.
2. Trabalhe em pequenos commits com mensagens diretas e descritivas.
3. Atualize a documentação sempre que adicionar uma funcionalidade ou alterar a estrutura.

## Estrutura de commits

- `feat:` nova funcionalidade
- `fix:` correção de bug
- `docs:` atualização de documentação
- `refactor:` refatoração sem mudança de comportamento
- `test:` adição ou correção de testes

## Controle de qualidade

- Mantenha o código legível e modular.
- Use `README.md` e `docs/` para explicar decisões e como executar o projeto.
- Verifique se os arquivos temporários e dependências não estão no repositório.

## Documentação e portfolio

- Para projetos de estudo: inclua seções de aprendizado, problemas resolvidos e decisões de design.
- Para hackathons: mantenha o MVP claro, destaque o valor e documente o que pode ser melhorado depois.
- Para portfólio: adicione descrições, capturas de tela, demos e URLs de deploy quando disponíveis.

## Pull Requests

- Descreva o objetivo e o impacto da alteração.
- Liste passos para testar localmente.
- Referencie issues ou tarefas relacionadas quando houver.
